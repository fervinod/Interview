 RSS vs VSZ
 These are memory usage metrics shown by commands like:

ps aux
top

RSS — Resident Set Size

Resident Set Size

RSS is the actual physical RAM currently used by a process


It includes:

loaded program code
stack
heap
shared libraries currently in RAM

Does NOT include:

swapped-out memory
memory not currently resident in RAM


VSZ — Virtual Memory Size

Virtual Memory Size

VSZ is the total virtual memory allocated to a process.

It includes:

RAM used
swapped memory
mapped files
shared libraries
reserved but unused memory

In TOP output command :

| Column | Meaning       |
| ------ | ------------- |
| VIRT   | VSZ           |
| RES    | RSS           |
| SHR    | Shared memory |


ps -eo pid,cmd,vsz,rss --sort=-rss | head

Memory Leak Clue

Possible leak pattern:

Metric	Behavior
RSS	Continuously increases
VSZ	May also increase

Growing RSS over time is more concerning.

. How would you diagnose memory leaks?

What indicates a leak:
Used memory keeps increasing steadily
Cache does NOT explain the growth
Swap usage starts increasing


11. Differentiate leak vs normal usage
Pattern	Meaning
RSS increases steadily forever	Memory leak
RSS grows then stabilizes	Normal behavior
Cache increases but MemAvailable stable	Not a leak
Swap increases + RSS grows	Severe issue

12. Common causes of memory leaks
Unreleased heap memory (C/C++)
Java heap misconfiguration
Too many open connections
File descriptor leaks
Cache not bounded
Kernel driver leaks (rare)

Summary

To diagnose memory leaks:

Monitor memory trend (free, vmstat)
Identify growing process (top, ps)
Track RSS over time (/proc, watch)
Use profiling tools (valgrind, jmap, smem)
Check kernel logs (dmesg, journalctl)
Confirm sustained growth = leak

14. Common root causes of server crash
Cause	Symptom
OOM Killer	process killed, system unstable
Disk failure	I/O errors, freeze
Kernel panic	system reboot
CPU overload	system hang
Memory leak	gradual slowdown
Filesystem full	services fail
Hardware issue	random reboot

To investigate a server crash:

Confirm reboot time (last reboot)
Analyze previous boot logs (journalctl -b -1)
Check kernel messages (dmesg)
Look for OOM kills
Check disk and hardware errors
Validate memory/CPU pressure
Review service logs
Check crash dumps (kdump)

a logical memory layer
created by the OS kernel

independent of physical RAM

Virtual memory is a memory management system where each process gets its own “private view” of memory,
instead of directly using physical RAM.
